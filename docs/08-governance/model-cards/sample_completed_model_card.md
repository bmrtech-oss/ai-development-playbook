## Intended Use

Providing concise, actionable code review feedback on pull requests within the AI Platform internal tools.

This model is designed to suggest review comments, identify likely style issues, and recommend small, safe edits for developer-facing code review contexts.

---

## Out-of-Scope Use

- Generating production code without human oversight.
- Analyzing proprietary third-party code without license compliance review.
- Security vulnerability scanning (use dedicated tooling instead).
- Making final approvals or gating merge decisions.

---

## Training Data & Provenance

Fine-tuned on 15,000 internal PR review comments that were anonymized and approved by governance. Augmented with 5,000 synthetic examples generated by GPT-4 to improve coverage for edge-case comment styles.

Data filtering included:
- PII removal from author names, emails, and proprietary identifiers.
- Toxic language filtering to remove abusive or non-professional phrasing.
- Normalization of code snippets to preserve structure without exposing secrets.

## Ethical Considerations & Bias

Red team identified a slight preference for verbose explanations over terse directives. Mitigation includes a system prompt that explicitly requests "concise, bullet-point style feedback."

Known limitation: the model may suggest variable name changes that conflict with the company style guide. A post-processing filter has been added to suppress suggestions that violate documented naming conventions.
