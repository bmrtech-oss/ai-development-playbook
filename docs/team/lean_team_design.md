# Lean Team Design

**Last Updated:** 2026-04-18

Our engineering team is structured for **flow efficiency**, not resource efficiency.

## Current Structure
- **Kernel Team (3‑4 engineers):** Platform core, SLM training, infrastructure.
- **Feature Teams (2‑3 engineers each):** VS Code extension, Web Dashboard.
- **Floating Roles:** Release Manager, Dev Productivity (rotate monthly).

## Principles
- **Small batch sizes:** No feature branch lives longer than 3 days.
- **T‑shaped skills:** Everyone can fix a frontend bug or tweak a prompt.
- **No handoffs:** The person who writes the code deploys it.

## Meeting Cadence
- Daily async standup (Geekbot).
- Weekly 30‑min sync (priority alignment).
- Monthly 1h deep‑dive (architecture / process).
